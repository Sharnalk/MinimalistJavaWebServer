A Minimalist Java Web Server
It works!

This is the default welcome page used to test the correct operation of the Java web server.

This project is not production ready. It was made for educational purposes.

If you can read this page, it means that the MinimalistJavaWebServer installed at this site is working properly. You should replace this file (located in MinimalistJavaWebServer/src/main/resources/default.html) before continuing to operate your HTTP server.

How to use ?

This server is able to serve your static files. Simply place your project folder alongside MinimalistJavaWebServer.jar, launch the server, and access the right URL. No external dependencies are needed, as all required libraries are included within Java.

You can organize your project folder as you want, but the recommended structure is as follows: 

MinimalistJavaWebServer/your_project/
                        |-- index.html # or the name you want
                        |
                        |-- static/
                        |         -- css/
                        |               style.css
                        |         -- img/
                        |               1.png
                        |               2.png
                        |               3.png
                        |         -- js/
                        |              index.js
  • When your project folder is ready, you can run the server with: 
    java -jar MinimalistJavaWebServer.jar
    The server listens on localhost:8080 by default, but you can specify another host/port when launching the server. The port number must be of type int, and the host must be a string.

    At runtime, specify the port and host directly as arguments to the JAR: 
    java -jar MinimalistJavaWebServer.jar --port 8000
  • Now the server is running, access the URL displayed in your terminal (host address + port number).
    You should see this welcome page unless you replaced default.html with your own content.
  • Remember to include your HTML file in the URL; otherwise, the server won't find it. For example:
    http://host:port/your_project/index.html
  • That's it! You should now be able to see your HTML file, and all linked static files will be served as well.
Note for Linux users

By default, Linux does not allow access through the web browser to any file apart of those located in /var/www, public_html directories (when enabled) and /usr/share (for web applications). If your site is using a web document root located elsewhere you may need to modify your files permissions accordingly.
